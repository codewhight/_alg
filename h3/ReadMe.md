暴力法的各種用途

何謂暴力法:
被稱為窮舉法（Exhaustive Search），是演算法中最直觀、最簡單，但也往往是最缺乏效率的一種解決問題策略。

簡單來說，它的核心概念就是：「把所有可能的答案全部試一遍，直到找到正確答案為止。」

運作步驟：
1.列舉 (Enumerate)： 產生該問題所有可能的候選解（Possibilities）。
2.檢查 (Check)： 逐一檢查每一個候選解是否符合問題的條件。
3.輸出 (Output)： 一旦找到符合的解，就輸出結果；如果全部試完都沒有，則宣告無解。

優缺點:
優點
    1.保證找到解答
        只要解存在，暴力法一定找得到。因為它採用窮舉策略，試過了所有的可能性，絕無漏網之魚。

    2.實作簡單
        邏輯非常直觀，不需要複雜的演算法設計或高深的資料結構。
        工程師撰寫程式的速度通常很快，適合快速原型開發 (Prototyping)。

    3.最佳基準測試
        它是用來驗證其他「高效演算法」是否正確的最佳對照組。
        在優化程式前，通常先寫一個暴力解來確保答案的正確性 (Correctness check)。

缺點 
    1.效率極低
        當資料規模 ($n$) 變大時，運算時間會呈**指數級爆炸成長**。
        常見的時間複雜度為 $O(n!)$ 或 $O(2^n)$，在大數據場景下幾乎不可行。

    2.資源消耗巨大
        對於複雜問題（例如：密碼破解、西洋棋佈局分析），其所需的計算量極大。
        在極端情況下，即便耗盡全宇宙的時間，電腦也無法跑完所有的可能性。
