說明 C(n,k) 的查表版和動態規劃版之異同

與AI的對話:https://gemini.google.com/share/cda0c4e4ab68

核心概念
查表法 (Top-Down)：從目標C(n, k) 開始，「由上而下」遞迴查詢。如果該點沒算過，就往下算 像是缺什麼才去拿什麼。

動態規劃 (Bottom-Up)：從C(0, 0)開始，「由下而上」一層層填滿表格，直到算到C(n, k)。像是 先把地基打好再蓋樓。

查表版 (Memoization / Top-Down)
是「遞迴」加上「快取」的變形。
程式會維護一個表格（通常是二維陣列或 Hash Map）。
邏輯： 每次呼叫函數時，先檢查表格內是否有值。有值： 直接返回（查表）。無值： 執行遞迴計算，將結果存入表格，然後返回。

優點： 只會計算「達成目標所需的必要狀態」。在某些稀疏圖或特殊條件下，計算量可能比填滿整張表少。
缺點： 遞迴深度過深時（例如 n 很大），會導致 Stack Overflow（堆疊溢位）。

動態規劃版 (Iterative DP / Bottom-Up)
利用迴圈直接填表。
邏輯： 建立一個表格，利用雙層迴圈，從 i=0 到n，由小到大算出每一項的值。

優點： 沒有遞迴呼叫的額外開銷。絕對不會發生 Stack Overflow。
空間優化容易：可以透過「滾動陣列（Rolling Array）」將空間複雜度從 $O(n \times k)$ 壓縮到 $O(k)$ 甚至 $O(1)$（視需求而定）。
缺點： 必定會算出 0 到 n 層的所有相關數值，無法跳過某些狀態。
